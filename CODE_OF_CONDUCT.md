# Code of Conduct

This page defines the participation expectations for issues, pull requests, discussions, and other project interactions.

YAi! values clear, professional, constructive participation.

Participants are expected to:

- be respectful and direct;
- focus on technical substance;
- avoid harassment, abuse, threats, or personal attacks;
- avoid submitting spam, low-effort AI-generated pull requests, or irrelevant content;
- respect project scope, maintainer time, and licensing requirements.

The maintainer may close issues, reject pull requests, or block participants who act in bad faith or repeatedly violate these expectations.
