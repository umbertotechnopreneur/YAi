# Documentation Taxonomy Foundation

## At a glance

- Started the governed YAi docs taxonomy with shared governance, history scaffolding, inbox routing, and project-local docs folders.
- Kept this record in shared history because it explains the repo-wide documentation classification model, not one project's local docs or one shared area's ongoing content.

## Summary

This migration starts the new documentation structure without immediately moving every existing spec pack.

The repo now has:

- a visible governance file,
- an always-on instructions file for agent behavior,
- a shared history area,
- a docs inbox,
- and project-local docs folders for implementation notes.

## Why this happened

The previous structure made it difficult to tell:

- which documents were still active,
- which ones were historical,
- which ones belonged to a single project,
- and which ones defined cross-project behavior.

## First migration rule

New docs should use the governed structure immediately.

Existing docs can move in phases so links and references are updated intentionally rather than through a large blind rename.
