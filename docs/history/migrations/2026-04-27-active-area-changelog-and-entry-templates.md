# Active Area Changelog and Entry Templates

## At a glance

- Added the active-area changelog and the shared entry template so the three-tier history model is complete and easier to use.
- Kept this record in shared history because it refined the history system across shared docs surfaces rather than changing just one area's content.

## Summary

This change adds two missing pieces to the documentation history model:

- an area-level changelog for `docs/specs/active/`
- a lightweight shared template file for future changelog entries

## Why

The operations and reference areas already had their own changelog files, but the active-spec surface still relied only on top-level history.

At the same time, the project-local and area-level changelogs needed a small shared writing guide so entries stay short and consistent instead of drifting into repeated recap text.

## Result

The docs system now has:

- a mirrored area-level changelog for active specs
- a shared singleton that defines lightweight changelog entry shapes
- a clearer path for future changelog updates at the project, shared-area, and top-level history layers
- a small routing table inside the template so writers can choose between project-local, shared-area, and top-level history more consistently
