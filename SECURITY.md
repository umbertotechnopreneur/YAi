# Security Policy

This page covers responsible disclosure for security issues in YAi!.

## Reporting Security Issues

If you discover a security issue in YAi!, do not open a public issue with exploit details.

Report it privately instead:

- Preferred email: [hello@umbertogiacobbi.biz](mailto:hello@umbertogiacobbi.biz)
- Website contact path: [https://umbertogiacobbi.biz/](https://umbertogiacobbi.biz/)

When possible, include:

- the affected version, commit, or branch
- clear reproduction steps
- expected impact and any known mitigations
- whether sensitive data, secrets, local files, or approval boundaries are involved

## Sensitive Information

Do not include secrets, API keys, tokens, credentials, private endpoints, customer data, or confidential business information in issues, pull requests, logs, screenshots, test fixtures, or discussions.

## Use The Right Channel

- Use this page for security vulnerabilities and trust-boundary issues.
- Use [SUPPORT.md](SUPPORT.md) for general product questions, build problems, and non-security support.

## Security Scope

YAi! focuses on local-first, explicit, auditable, approval-based agent execution. Changes touching filesystem operations, shell execution, workflow execution, approvals, audit, memory mutation, or external integrations should be reviewed carefully.
